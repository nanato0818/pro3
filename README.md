# pro3 — トレーニングルーム予約システム

学校のトレーニングルームを生徒がスマホ・PCから予約し、入退室の時間を記録・管理できるWebアプリです(pro3の授業課題)。

**公開デモ**: https://hiiragi.pythonanywhere.com (スマホ・PCのブラウザからそのまま利用できます)

## 主な機能

### 利用者向け
- クラス学年(I1〜I5 / M1〜M5 / S1〜S5)・出席番号・名前による簡易登録/ログイン(パスワードなし)
- 月間カレンダーからの予約(開始・終了時刻を15分刻みで指定、6:00〜22:00)
- 週7時間までの予約上限チェック(前週に超過があると、その分だけ翌週の上限が自動で減額される**超過ペナルティ**付き)
- 同時間帯の**定員制限**(既定20名。満員の時間帯は予約不可)
- 予約一覧表示・取り消し(今日以降の有効な予約のみ表示)
- 入室〜退室のトレーニングセッション管理(円形リングの経過時間タイマー、予定時間超過時は警告表示+ブラウザ通知、超過時間を記録)
- 予約開始15分前のリマインダー通知(ブラウザ通知)
- 週間レポート(日別・週別のトレーニング時間グラフ、超過分は赤で表示)

### 管理者向け
- 登録時に管理者コードを入力すると管理者になる(コードは本番環境では環境変数 `ADMIN_CODE` で上書き)
- 管理者ダッシュボード: 本日の全予約(強制取り消し可)/ユーザー一覧(管理者権限の付与・剥奪)/全員のトレーニング実績(超過は赤で強調)
- 定員数の変更
- 実績のCSVエクスポート(Excel対応のBOM付きUTF-8)

## UI

スマホ最適化のモバイルアプリ風デザイン(ダークテーマ)。画面下部の固定タブバーで
「ホーム / 予約 / レポート / 管理(管理者のみ)」を切り替えます。

## 技術構成

- **バックエンド**: Python 3.11 + Flask(軽量Webフレームワーク)
- **データベース**: SQLite(users / reservations / training_logs / settings の4テーブル。初回起動時に自動作成)
- **フロントエンド**: Jinja2テンプレート + vanilla JS(外部ライブラリ不使用)+ CSS(フレームワーク不使用)
- **ホスティング**: PythonAnywhere(無料プラン)

## セットアップ手順(ローカル実行)

```
pip install -r requirements.txt
python app.py
```

起動後、ブラウザで http://localhost:5000 にアクセスしてください。
初回起動時に `training.db`(SQLiteデータベース)が自動生成されます(Git管理外)。

## ディレクトリ構成

```
pro3/
├── app.py             # Flaskアプリ本体(ルート定義・予約/定員/ペナルティ等のロジック)
├── database.py        # DB接続・スキーマ作成(後方互換マイグレーション付き)
├── templates/         # Jinja2テンプレート(home / calendar / reserve / session / report / admin ほか)
├── static/
│   ├── style.css      # デザインシステム(ダークテーマ、CSS変数で統一)
│   └── app.js         # タイマー・リマインダー通知・確認ダイアログ
├── requirements.txt
└── training.db        # 実行時に自動生成(Git管理外)
```

## 設計上の割り切り(既知の制限)

- 認証はクラス+出席番号のみの簡易方式(パスワードなし)。課題スコープの割り切りであり、実運用にはパスワード等の本人確認が必要
- ブラウザ通知はページを開いている間のみ動作(アプリを閉じていても届くWeb Pushはスコープ外)
